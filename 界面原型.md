# 台球厅会员管理系统 - 界面原型设计
以下界面原型基于“极简操作、核心功能优先”原则设计，覆盖主页、会员管理、充值、消费4大核心场景，适配Windows桌面端（分辨率1366×768及以上），风格简洁清晰，降低运营者学习成本。


## 一、整体设计规范
1. **布局结构**：采用“顶部导航栏+左侧功能栏+中部内容区”固定布局，导航/功能栏常驻，内容区随功能切换；  
2. **色彩方案**：主色#2E86AB（蓝色，体现专业稳定），辅助色#A23B72（紫色，用于按钮/强调元素），背景色#F8F9FA（浅灰，降低视觉疲劳）；  
3. **交互逻辑**：所有操作按钮hover时变色，表单提交前做合法性校验，操作成功/失败弹出 Toast 提示（如“充值成功”“余额不足”）；  
4. **字体规范**：标题用微软雅黑16px（加粗），正文用微软雅黑14px，辅助文字（如备注）用微软雅黑12px（灰色）。


## 二、核心界面原型设计
### 1. 主页（数据概览页）
#### 1.1 界面布局
| 区域          | 内容与交互设计                                                                 |
|---------------|------------------------------------------------------------------------------|
| **顶部导航栏** | - 左侧显示系统名称“台球厅会员管理系统”；<br>- 右侧显示当前登录操作员（如“操作员：张三”）+“退出登录”按钮。 |
| **数据卡片区** | 2个核心数据卡片横向排列，占内容区上半部分：<br>- **近7日充值总额**：<br>  - 顶部大数字（如“¥3,200”）+ 小字“近7日”；<br>  - 下方折线图（X轴：日期，Y轴：单日充值金额），悬停显示“10-25：¥500”；<br>  - 卡片右下角“查看明细”按钮，点击跳转充值记录列表；<br>- **近7日消费总额**：<br>  - 结构同上，大数字（如“¥2,800”）+ 折线图，“查看明细”按钮跳转消费记录列表。 |

### 2. 会员管理页（含列表与信息操作）
#### 2.1 会员列表页
| 区域          | 内容与交互设计                                                                 |
|---------------|------------------------------------------------------------------------------|
| **左侧功能栏** | 固定显示功能菜单，当前选中“会员管理”：<br>- 主页<br>- 会员管理（√ 选中）<br>- 充值操作<br>- 消费操作<br>- 充值记录<br>- 消费记录 |
| **顶部操作区** | - 左侧“会员列表”标题；<br>- 右侧两个按钮：“添加会员”（紫色背景，点击弹出新增表单）、“刷新列表”（灰色背景，点击重新加载列表）；<br>- 搜索框： placeholder“输入姓名/手机号/会员编号”，输入时实时模糊筛选列表。 |
| **会员列表区** | 表格形式展示，字段横向排列，表头固定，内容可滚动：<br>- 表头：会员编号、姓名、手机号、余额（元）、注册时间、操作；<br>- 数据行：<br>  - 手机号脱敏（如“138****1234”）；<br>  - 余额显示（如“¥500.00”）；<br>  - 操作列：“详情”“编辑”“删除”三个按钮（图标+文字，如🖱️ 详情）；<br>- 分页：底部显示“共32条 1/2页”，+ 页码按钮（1、2）+ “上一页/下一页”，点击切换。 |

#### 2.2 新增/编辑会员表单（弹窗）
- **弹窗标题**：新增会员 / 编辑会员（根据操作切换）；  
- **表单字段**：  
  1. 姓名：输入框（必填，placeholder“请输入会员姓名”，校验非空）；  
  2. 手机号：输入框（必填，placeholder“请输入11位手机号”，校验11位数字）；  
  3. 余额：只读文本（显示“¥0.00”或当前余额，不可编辑）；  
- **底部按钮**：“取消”（灰色，关闭弹窗）、“确认”（紫色，提交表单，校验通过后关闭弹窗并刷新列表）。

#### 2.3 会员详情页（弹窗）
- **弹窗标题**：会员详情（编号：8001）；  
- **详情内容**：分2块展示：  
  1. 基础信息：<br>  - 姓名：张三<br>  - 手机号：138****1234<br>  - 余额：¥500.00<br>  - 注册时间：2025-10-01 14:30；  
  2. 最近记录：<br>  - 标题“最近3条记录”；<br>  - 表格：类型（充值/消费）、金额、操作人、时间（如“充值 ¥200 张三 2025-10-25 10:15”）；  
- **底部按钮**：“关闭”（灰色，关闭弹窗）。


### 3. 充值操作页
#### 3.1 界面布局
| 区域          | 内容与交互设计                                                                 |
|---------------|------------------------------------------------------------------------------|
| **左侧功能栏** | 当前选中“充值操作”（同会员管理页菜单）。                                      |
| **顶部标题区** | 标题“会员充值” + 副标题“选择会员后输入充值金额”。                           |
| **会员选择区** | 搜索+选择组合：<br>- 搜索框：placeholder“输入会员编号/手机号/姓名搜索”，输入时下拉显示匹配会员列表（如“8001 张三 138****1234”）；<br>- 选中会员后，下方显示会员信息卡片：<br>  - 会员编号：8001<br>  - 姓名：张三<br>  - 当前余额：¥300.00。 |
| **充值表单区** | 表单居中显示，仅1个核心字段：<br>- 充值金额：输入框（必填，placeholder“请输入充值金额（元）”，校验“正数、非零、最多2位小数”，如输入“200”）；<br>- 实时计算：输入后下方显示“充值后余额：¥500.00”（当前余额+充值金额）；<br>- 操作人：只读文本（显示当前登录操作员，如“操作人：张三”）。 |
| **底部按钮区** | 两个按钮居中排列：<br>- “取消”（灰色，点击返回主页）；<br>- “确认充值”（紫色，点击后校验：① 已选会员 ② 金额合法，通过后弹出Toast“充值成功”，重置表单并跳转主页）。 |


### 4. 消费操作页
#### 4.1 界面布局（与充值页结构相似，差异点标注）
| 区域          | 内容与交互设计（重点标注与充值页差异）                                       |
|---------------|------------------------------------------------------------------------------|
| **顶部标题区** | 标题“会员消费” + 副标题“选择会员后输入消费金额”。                           |
| **会员选择区** | 与充值页一致：搜索选择会员后，显示会员信息卡片（含当前余额）。              |
| **消费表单区** | 核心差异点：<br>- 消费金额：输入框（校验“正数、非零、最多2位小数”，且“金额≤当前余额”，如当前余额¥500，输入“150”）；<br>- 实时计算：输入后显示“消费后余额：¥350.00”（当前余额-消费金额）；<br>- 新增“密码验证”字段：输入框（placeholder“请输入会员密码（初始为手机号后6位）”，必填，校验正确后才可提交）。 |
| **底部按钮区** | 按钮文字调整：<br>- “取消”（灰色，返回主页）；<br>- “确认扣款”（紫色，点击后校验：① 已选会员 ② 金额合法（≤余额） ③ 密码正确，通过后弹出Toast“扣款成功”，重置表单并跳转主页；校验失败提示“余额不足”“密码错误”）。 |


### 5. 充值/消费记录页（以充值记录为例，消费记录结构一致）
#### 5.1 界面布局
| 区域          | 内容与交互设计                                                                 |
|---------------|------------------------------------------------------------------------------|
| **左侧功能栏** | 当前选中“充值记录”（或“消费记录”）。                                          |
| **顶部操作区** | - 标题“充值记录列表”；<br>- 筛选框：日期范围（默认“近7日”，可选择“今日”“近30日”或自定义日期区间）+ 会员搜索（placeholder“搜索会员编号/姓名”）；<br>- “导出Excel”按钮（灰色，点击导出当前筛选条件下的记录）。 |
| **记录列表区** | 表格形式，字段：<br>- 充值ID、会员编号、会员姓名、充值金额（¥）、操作人、操作时间、备注（可选，如“主动充值”）；<br>- 分页：同会员列表页（底部页码导航）；<br>- 无数据时显示“暂无充值记录”（居中灰色文字）。 |


## 三、界面跳转逻辑
1. **主页 → 其他页面**：点击左侧功能栏菜单跳转（如主页→会员管理、主页→充值操作）；  
2. **会员管理 → 充值/消费**：在会员列表点击“详情”，弹窗内提供“去充值”“去消费”按钮，点击跳转对应操作页并自动带入会员信息；  
3. **充值/消费 → 主页**：操作成功后自动跳转主页，或点击“取消”手动返回主页；  
4. **数据明细跳转**：主页点击“查看明细”，跳转充值/消费记录页并默认筛选“近7日”数据。


该原型设计聚焦“少操作、高清晰”，所有核心功能均可在3步内完成（如“选会员→输金额→确认充值”），符合台球厅运营场景下“快速操作、减少等待”的需求。
